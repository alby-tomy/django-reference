What is Django?

Foundation

Django is a high-level Python web framework that follows the Model-View-Template (MVT) pattern. "Batteries-included" means routing, ORM, admin, auth, forms, and templating ship out of the box — no picking an ecosystem, everything works together.

The MVT Architecture

Django calls its pattern MVT, but it maps to MVC like this:

HTTP Request
→
urls.py
Router
→
views.py
View (Controller)
→
models.py
Model (DB)
↓
template.html
Template
→
Browser
HTTP Response
Model

Defines your data structure. Django's ORM maps Python classes to database tables. You write Python, Django handles SQL.

View

Business logic. Receives a request, queries the database, prepares a context dict, returns a response. The "controller" of MVC.

Template

HTML with Django template tags. Receives the context dict from the view and renders it into a final HTML string for the browser.

💡 Django vs Flask vs FastAPI

Django — full-stack, batteries included, best for traditional web apps with admin, auth, ORM.
Flask — micro-framework, minimal by default, you choose every library yourself.
FastAPI — async-first, best for pure APIs with auto-generated OpenAPI docs.

Setup
Terminal — complete setupBash

python -m venv venv
source venv/bin/activate        # Linux/macOS
# venv\Scripts\activate         # Windows

pip install django               # installs latest stable
pip install django==4.2          # pin a specific version
python -m django --version       # verify: prints 4.x or 5.x

# ── 3. Create a Django PROJECT ──────────────────────────────

django-admin startproject mysite .    # '.' = put in current dir

# ── 4. Create an APP inside the project ─────────────────────
# An app = self-contained module (blog, shop, accounts, etc.)
python manage.py startapp blog

# ── 5. Run initial migrations & dev server ──────────────────
python manage.py migrate         # creates db.sqlite3
python manage.py createsuperuser # admin login
python manage.py runserver       # http://127.0.0.1:8000/

# ── 6. Freeze dependencies ──────────────────────────────────
pip freeze > requirements.txt
pip install -r requirements.txt  # restore on another machine
⚠️ Project vs App — Critical Distinction

Project = the whole website. Created once with django-admin startproject. Contains global config.
App = a self-contained feature module (blog, payments, accounts). One project can have many apps.
Register every new app in INSTALLED_APPS in settings.py, otherwise Django ignores it entirely.

What Each File Actually Does

manage.py

A thin wrapper around Django's command-line utility. Every management command (runserver, migrate, shell, createsuperuser) goes through this. Never edit it.

settings.py

The brain. Controls everything: which database, which apps are installed, where templates live, how auth works, email config, caching, static files. Every framework feature reads from here.

urls.py (project)

The root URL dispatcher. Receives all incoming URLs and routes them to the right app's urls.py using include(). Think of it as a phone switchboard.

models.py

Define your data as Python classes that extend models.Model. Django auto-generates database tables from these. All DB operations go through the ORM — no raw SQL needed.

views.py

Receives HTTP requests, runs business logic, queries models, passes context to templates, returns HTTP responses. The controller layer. Can be functions (FBV) or classes (CBV).

admin.py

Register your models here and Django generates a full CRUD admin interface at /admin/ — zero extra code. Customise list displays, filters, search, inline editing.

migrations/

Auto-generated Python files that record every change to your models as versioned database migrations. Never edit manually. Run makemigrations + migrate after any model change.

wsgi.py / asgi.py

Server gateways. wsgi.py is for traditional sync servers (Gunicorn, uWSGI). asgi.py is for async servers (Daphne, Uvicorn) when you need WebSockets or async views.

Routing

blog/urls.py — app-level patternsPython
from django.urls import path, re_path
from . import views

urlpatterns = [

    # path() — exact matches
    path('', views.home, name='home'),                         # /
    path('about/', views.about, name='about'),                 # /about/

    # <type:param> — URL converters capture values into view kwargs
    path('articles/<int:pk>/', views.detail, name='detail'),   # /articles/42/
    path('articles/<slug:slug>/', views.detail_slug, name='detail-slug'),
    path('users/<str:username>/', views.profile, name='profile'),
    path('files/<path:file_path>/', views.serve_file, name='file'),

    # re_path() — regex for complex patterns
    re_path(r'^archive/(?P<year>\d{4})/(?P<month>\d{2})/$', views.archive),

    # Class-based views need .as_view()
    path('new/', views.ArticleCreateView.as_view(), name='create'),
    path('<pk>/edit/', views.ArticleUpdateView.as_view(), name='update'),

]

# URL converters: int, str, slug, uuid, path
# int   → only digits, converted to Python int
# str   → any non-empty non-slash string (default)
# slug  → ASCII letters, numbers, hyphens, underscores
# uuid  → UUID formatted string → uuid.UUID instance
# path  → any non-empty string INCLUDING slashes

Function-Based Views (FBV)

Views

The simplest Django view: a Python function that takes an HttpRequest and returns an HttpResponse. Explicit, readable, easy to debug. You control every step.

blog/views.py — FBV patternsPython
from django.shortcuts import render, get_object_or_404, redirect
from django.http import HttpResponse, Http404, JsonResponse
from django.contrib.auth.decorators import login_required
from django.views.decorators.http import require_http_methods, require_POST
from .models import Article
from .forms import ArticleForm


# ── 1. Simplest view ─────────────────────────────────────────
def home(request):
    articles = Article.objects.filter(published=True).order_by('-created_at')
    context = {'articles': articles, 'page_title': 'Home'}
    return render(request, 'blog/home.html', context)


# ── 2. Detail view with URL parameter ────────────────────────
def article_detail(request, slug):
    # get_object_or_404: returns object or renders 404 page
    article = get_object_or_404(Article, slug=slug, published=True)
    return render(request, 'blog/detail.html', {'article': article})


# ── 3. Handling GET + POST in one view ───────────────────────
def article_create(request):
    if request.method == 'POST':
        form = ArticleForm(request.POST, request.FILES)  # FILES for uploads
        if form.is_valid():
            article = form.save(commit=False)          # don't save yet
            article.author = request.user              # attach current user
            article.save()
            return redirect('blog:detail', slug=article.slug)
    else:
        form = ArticleForm()  # empty form on GET

    return render(request, 'blog/create.html', {'form': form})


# ── 4. Login required decorator ──────────────────────────────
@login_required  # redirects to LOGIN_URL if not authenticated
def dashboard(request):
    my_articles = Article.objects.filter(author=request.user)
    return render(request, 'blog/dashboard.html', {'articles': my_articles})


# ── 5. HTTP method restriction ────────────────────────────────
@require_http_methods(['GET', 'POST'])  # returns 405 for other methods
def contact(request):
    if request.method == 'POST':
        # process form
        return redirect('blog:home')
    return render(request, 'blog/contact.html')


# ── 6. JSON API view ─────────────────────────────────────────
def api_articles(request):
    articles = Article.objects.filter(published=True).values(
        'id', 'title', 'slug', 'created_at'
    )
    return JsonResponse({'articles': list(articles)})


# ── 7. Delete with confirmation ───────────────────────────────
@login_required
@require_POST  # only allow POST (from a confirm form)
def article_delete(request, pk):
    article = get_object_or_404(Article, pk=pk, author=request.user)
    article.delete()
    return redirect('blog:home')


# ── 8. Decorator stacking ────────────────────────────────────
# Decorators apply bottom-up; login_required runs first here
@login_required
@require_http_methods(['GET', 'POST'])
def protected_create(request):
    # Only authenticated users with GET or POST reach this code
    pass

Class-Based Views (CBV)

Views

Class-Based Views organise view logic into methods per HTTP method (get(), post()). The base View class handles dispatch. You can override only what differs.

blog/views.py — CBV from scratchPython
from django.views import View
from django.shortcuts import render, redirect, get_object_or_404
from django.contrib.auth.mixins import LoginRequiredMixin, PermissionRequiredMixin
from .models import Article
from .forms import ArticleForm


# ── Base View ────────────────────────────────────────────────
class ArticleListView(View):
    """Get method handles GET requests, post() handles POST, etc."""
    template_name = 'blog/list.html'

    def get(self, request, *args, **kwargs):
        articles = Article.objects.filter(published=True)
        return render(request, self.template_name, {'articles': articles})


# ── Create view with GET + POST ───────────────────────────────
class ArticleCreateView(LoginRequiredMixin, View):
    # LoginRequiredMixin: redirects to login if not authenticated
    template_name = 'blog/create.html'
    login_url = '/accounts/login/'   # override default login URL

    def get(self, request):
        form = ArticleForm()
        return render(request, self.template_name, {'form': form})

    def post(self, request):
        form = ArticleForm(request.POST, request.FILES)
        if form.is_valid():
            article = form.save(commit=False)
            article.author = request.user
            article.save()
            return redirect('blog:detail', slug=article.slug)
        return render(request, self.template_name, {'form': form})


# ── Mixins for reusable behaviour ─────────────────────────────
class AdminView(LoginRequiredMixin, PermissionRequiredMixin, View):
    permission_required = 'blog.can_publish'  # custom permission
    raise_exception = True                    # raise 403 instead of redirect

    def get(self, request):
        return render(request, 'blog/admin.html')


# ── urls.py: CBV requires .as_view() ─────────────────────────
# path('', ArticleListView.as_view(), name='list')
# path('new/', ArticleCreateView.as_view(), name='create')
🔑 dispatch() — How CBVs Work Internally

When Django calls View.as_view(), it creates a callable. On each request, dispatch() reads request.method and calls the matching method (self.get(), self.post(), etc.). If no matching method exists, it returns 405 Method Not Allowed. You can override dispatch() to add logic that runs before any method.

Generic Class-Based Views

Views

Django ships generic CBVs that handle common patterns (list, detail, create, update, delete) out of the box. Minimal code — maximum power. Set a few class attributes and you're done.

blog/views.py — all 5 generic viewsPython
from django.views.generic import (
    ListView, DetailView,
    CreateView, UpdateView, DeleteView
)
from django.contrib.auth.mixins import LoginRequiredMixin
from django.urls import reverse_lazy
from .models import Article


# ── ListView — renders a list of objects ─────────────────────
# Auto context: object_list (or 'article_list')
# Default template: blog/article_list.html
class ArticleListView(ListView):
    model = Article
    template_name = 'blog/home.html'       # override default
    context_object_name = 'articles'       # rename from 'object_list'
    ordering = ['-created_at']             # sort order
    paginate_by = 10                        # adds pagination automatically

    def get_queryset(self):
        # Override to filter or add custom logic
        return Article.objects.filter(published=True).select_related('author')

    def get_context_data(self, **kwargs):
        # Add extra variables to the template context
        ctx = super().get_context_data(**kwargs)
        ctx['page_title'] = 'All Articles'
        ctx['total'] = self.get_queryset().count()
        return ctx


# ── DetailView — renders one object ──────────────────────────
# Fetches by pk or slug from URL kwargs automatically
# Default template: blog/article_detail.html
class ArticleDetailView(DetailView):
    model = Article
    template_name = 'blog/detail.html'
    context_object_name = 'article'
    slug_url_kwarg = 'slug'              # which URL kwarg has the slug

    def get_object(self):
        obj = super().get_object()
        # Custom logic: increment view count
        Article.objects.filter(pk=obj.pk).update(views=obj.views + 1)
        return obj


# ── CreateView — show form + save new object ──────────────────
# Default template: blog/article_form.html
class ArticleCreateView(LoginRequiredMixin, CreateView):
    model = Article
    fields = ['title', 'body', 'tags', 'cover_image']
    template_name = 'blog/form.html'
    success_url = reverse_lazy('blog:home')  # redirect after save

    def form_valid(self, form):
        # Runs when form passes validation — attach author before save
        form.instance.author = self.request.user
        return super().form_valid(form)


# ── UpdateView — edit existing object ────────────────────────
class ArticleUpdateView(LoginRequiredMixin, UpdateView):
    model = Article
    fields = ['title', 'body', 'tags']
    template_name = 'blog/form.html'

    def get_queryset(self):
        # Only allow author to edit their own articles
        return super().get_queryset().filter(author=self.request.user)

    def get_success_url(self):
        return reverse_lazy('blog:detail', kwargs={'slug': self.object.slug})


# ── DeleteView — confirm + delete ─────────────────────────────
# Default template: blog/article_confirm_delete.html
class ArticleDeleteView(LoginRequiredMixin, DeleteView):
    model = Article
    success_url = reverse_lazy('blog:home')
    template_name = 'blog/confirm_delete.html'

    def get_queryset(self):
        return super().get_queryset().filter(author=self.request.user)

FBV vs CBV — When to Use Which

Decision
CriterionFunction-Based View (FBV)Class-Based View (CBV)
Readability✓ Explicit, linear, easy to followRequires knowing Django's class hierarchy
BoilerplateMore repetition for CRUD✓ Generic CBVs eliminate all boilerplate
Custom logic✓ Easier — just write codeMust know which method to override
Reuse / DRYRequires decorators or helper functions✓ Mixins stack cleanly
Decorators✓ Just stack on the functionmethod_decorator or dispatch() override
API views✓ Great with DRF's @api_view✓ Great with DRF's APIView / ViewSets
Standard CRUDWrite the same pattern repeatedly✓ Use ListView/DetailView/CreateView etc.
Debugging✓ Stacktraces point to your code directlyStack goes through Django's generic internals
✓ Use FBV when
  • You're new to Django
  • The logic is highly custom or doesn't fit generic patterns
  • You need complex conditional flows
  • Writing API endpoints with @api_view
  • You want maximum readability for the team
✓ Use CBV when
  • Standard CRUD pages (list, detail, create, update, delete)
  • You need reusable behaviour via mixins
  • Building DRF ViewSets for APIs
  • Working in a large codebase with established patterns
  • Paginated list views
💡 The Django community consensus (2024)

FBVs for complex one-off views, custom logic, and simple pages. Generic CBVs for standard CRUD. DRF ViewSets for APIs. Avoid raw CBV (plain View subclass) for simple cases — FBV is cleaner. Mix freely — Django doesn't care.

Data

Models are Python classes that map to database tables. Every field type, relationship, and constraint you define here becomes a column in SQL. Django's ORM translates Python method calls into SQL queries automatically.

from django.db import models
from django.contrib.auth import get_user_model
from django.urls import reverse
from django.utils.text import slugify
from django.utils import timezone

User = get_user_model()  # always use this, not auth.User directly

class Article(models.Model):

    # ── STATUS CHOICES ───────────────────────────────────────────
    class Status(models.TextChoices):
        DRAFT     = 'draft',     'Draft'
        PUBLISHED = 'published', 'Published'
        ARCHIVED  = 'archived',  'Archived'

    # ── FIELD TYPES ──────────────────────────────────────────────
    title       = models.CharField(max_length=200)                   # VARCHAR(200)
    slug        = models.SlugField(max_length=200, unique=True)     # for URLs
    body        = models.TextField()                                  # TEXT (unlimited)
    excerpt     = models.TextField(blank=True)                       # optional
    cover       = models.ImageField(upload_to='covers/%Y/%m/',      # upload path
                                    blank=True, null=True)
    views       = models.PositiveIntegerField(default=0)
    featured    = models.BooleanField(default=False)
    status      = models.CharField(max_length=10,
                                   choices=Status.choices,
                                   default=Status.DRAFT)
    reading_min = models.PositiveSmallIntegerField(default=1)
    meta_json   = models.JSONField(default=dict, blank=True)        # JSON column

    # ── DATE FIELDS ──────────────────────────────────────────────
    created_at  = models.DateTimeField(auto_now_add=True)   # set once on creation
    updated_at  = models.DateTimeField(auto_now=True)        # updated on every save
    published_at = models.DateTimeField(null=True, blank=True)

    # ── RELATIONSHIPS ────────────────────────────────────────────
    # ForeignKey (many-to-one): many articles → one author
    author   = models.ForeignKey(User, on_delete=models.CASCADE,
                                 related_name='articles')
    # ForeignKey with nullable: article can have no category
    category = models.ForeignKey(Category, on_delete=models.SET_NULL,
                                 null=True, blank=True,
                                 related_name='articles')
    # ManyToManyField: article can have multiple tags
    tags     = models.ManyToManyField('Tag', blank=True,
                                      related_name='articles')

    # ── on_delete OPTIONS ────────────────────────────────────────
    # CASCADE    → delete this row when related object is deleted
    # SET_NULL   → set FK to NULL (field must allow null=True)
    # SET_DEFAULT→ set FK to field's default
    # PROTECT    → raise ProtectedError (prevents deletion)
    # RESTRICT   → similar to PROTECT but smarter
    # DO_NOTHING → do nothing (risk of IntegrityError)

    # ── META ─────────────────────────────────────────────────────
    class Meta:
        ordering = ['-published_at']           # default sort
        indexes = [
            models.Index(fields=['slug']),     # faster slug lookups
            models.Index(fields=['status', '-published_at']),
        ]
        constraints = [
            models.UniqueConstraint(
                fields=['author', 'slug'],
                name='unique_author_slug'
            )
        ]
        verbose_name = 'Article'
        verbose_name_plural = 'Articles'
        permissions = [('can_publish', 'Can publish articles')]

    # ── METHODS ──────────────────────────────────────────────────
    def __str__(self):
        return self.title

    def get_absolute_url(self):
        return reverse('blog:detail', kwargs={'slug': self.slug})

    def save(self, *args, **kwargs):
        if not self.slug:
            self.slug = slugify(self.title)
        if self.status == self.Status.PUBLISHED and not self.published_at:
            self.published_at = timezone.now()
        super().save(*args, **kwargs)

    @property
    def is_published(self):
        return self.status == self.Status.PUBLISHED

    @classmethod
    def published(cls):
        return cls.objects.filter(status=cls.Status.PUBLISHED)

# ── Style 1: Function-based (simple) ─────────────────────────
def timing_middleware(get_response):
    # Code here runs ONCE at startup

    def middleware(request):
        start = time.time()

        response = get_response(request)  # call the next layer / view

        duration = time.time() - start
        response['X-Response-Time'] = f'{duration:.3f}s'
        logger.info(f'{request.path} took {duration:.3f}s')

        return response

    return middleware


# ── Style 2: Class-based (more hooks) ────────────────────────
class MaintenanceModeMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response  # runs once at startup

    def __call__(self, request):
        # ── BEFORE view ──────────────────────────────────────────
        # Check maintenance mode setting
        from django.conf import settings
        if getattr(settings, 'MAINTENANCE_MODE', False):
            if not request.user.is_staff:
                return HttpResponse(
                    '<h1>Down for maintenance</h1>',
                    status=503,
                    content_type='text/html'
                )

        response = self.get_response(request)

        # ── AFTER view ───────────────────────────────────────────
        response['X-Content-Type-Options'] = 'nosniff'
        return response

    # Optional hooks (class-based only):
    def process_view(self, request, view_func, view_args, view_kwargs):
        # Called just before view is called; return None to continue
        pass

    def process_exception(self, request, exception):
        # Called if view raises an exception
        logger.error(f'View exception: {exception}', exc_info=True)

    def process_template_response(self, request, response):
        # Called if view returns TemplateResponse
        return response

                                       context={'request': request})
        if serializer.is_valid():
            serializer.save()

        return Response(serializer.errors, status=400)

from rest_framework import serializers
from django.contrib.auth.password_validation import validate_password
from .models import User


class RegisterSerializer(serializers.ModelSerializer):
    password  = serializers.CharField(write_only=True, validators=[validate_password])
    password2 = serializers.CharField(write_only=True, label='Confirm password')

    class Meta:
        model  = User
        fields = ['email', 'username', 'password', 'password2',
                  'first_name', 'last_name']

    # ── Cross-field validation ─────────────────────────────────
    def validate(self, attrs):
        if attrs['password'] != attrs.pop('password2'):
            raise serializers.ValidationError({'password2': 'Passwords do not match.'})
        return attrs

    # ── Custom field-level validation ──────────────────────────
    def validate_email(self, value):
        if User.objects.filter(email__iexact=value).exists():
            raise serializers.ValidationError('This email is already registered.')
        return value.lower()

    # ── Custom create ──────────────────────────────────────────
    def create(self, validated_data):
        return User.objects.create_user(**validated_data)   # hashes password


class UserSerializer(serializers.ModelSerializer):

class TaskViewSet(viewsets.ModelViewSet):
    """
    Full CRUD + custom actions for Task.
    GET    /tasks/          → list (TaskListSerializer, paginated, filtered)
    POST   /tasks/          → create (TaskDetailSerializer)
    GET    /tasks/{id}/     → retrieve (TaskDetailSerializer)
    PUT    /tasks/{id}/     → update
    PATCH  /tasks/{id}/     → partial update
    DELETE /tasks/{id}/     → destroy
    POST   /tasks/{id}/move/     → custom action: move to another project
    GET    /tasks/{id}/activity/ → custom action: audit log
    """
    permission_classes  = [permissions.IsAuthenticated, IsProjectMember]
    filterset_class     = TaskFilter           # django-filter class
    search_fields       = ['title', 'description', 'tags']
    ordering_fields     = ['priority', 'due_date', 'created_at', 'status']
    ordering            = ['-priority', 'due_date']

    def get_queryset(self):
        # Scope to the project from the URL; optimise with annotations
        project_id = self.kwargs.get('project_pk')
        return (
            Task.objects
            .filter(project_id=project_id)
            .select_related('assignee', 'reporter', 'project')
            .prefetch_related(
                'tags',
                Prefetch('comments', queryset=Comment.objects.select_related('author')),
                Prefetch('attachments', queryset=Attachment.objects.select_related('uploaded_by')),
                Prefetch('subtasks', queryset=Task.objects.select_related('assignee')),
            )
            .annotate(comment_count=Count('comments', distinct=True))
        )

    def get_serializer_class(self):
        # Use lightweight serializer for list, detailed for everything else
        if self.action == 'list':
            return TaskListSerializer
        return TaskDetailSerializer

    def get_serializer_context(self):
        ctx = super().get_serializer_context()
        # Pass project to serializer for assignee validation
        project_id = self.kwargs.get('project_pk')
        if project_id:
            ctx['project'] = get_object_or_404(Project, pk=project_id)
        return ctx

    def perform_create(self, serializer):
        project = get_object_or_404(Project, pk=self.kwargs['project_pk'])
        task = serializer.save(project=project, reporter=self.request.user)
        logger.info('task_created', extra={'task_id': str(task.id), 'project_id': str(project.id)})
        # Async email if assignee set
        if task.assignee_id:
            send_task_assignment_email.delay(str(task.id))

    def perform_update(self, serializer):
        old_assignee = serializer.instance.assignee_id
        task = serializer.save()
        # Send email only when assignee changes
        if task.assignee_id and task.assignee_id != old_assignee:
            send_task_assignment_email.delay(str(task.id))

    # ── Custom action: move task to different project ─────────
    @action(detail=True, methods=['post'], url_path='move')
    def move(self, request, pk=None, project_pk=None):
        task = self.get_object()
        new_project_id = request.data.get('project_id')
        new_project = get_object_or_404(
            Project, pk=new_project_id,
            members=request.user    # user must be member of destination
        )
        task.project = new_project
        task.save(update_fields=['project', 'updated_at'])
        logger.info('task_moved', extra={
            'task_id': str(task.id),
            'to_project': str(new_project.id)
        })
        return Response(TaskDetailSerializer(task, context=self.get_serializer_context()).data)

    # ── Custom action: bulk status update ─────────────────────
    @action(detail=False, methods=['post'], url_path='bulk-update')
    def bulk_update(self, request, project_pk=None):
        ids    = request.data.get('ids', [])
        status = request.data.get('status')
        if not ids or not status:
            return Response({'detail': 'ids and status required.'}, status=400)
        updated = self.get_queryset().filter(id__in=ids).update(status=status)
        return Response({'updated': updated})

from .models import Task


class TaskFilter(django_filters.FilterSet):
    # Exact match filters (auto-generated from field choices)
    status   = django_filters.MultipleChoiceFilter(choices=Task.Status.choices)
    priority = django_filters.MultipleChoiceFilter(choices=Task.Priority.choices)

    # Date range filters
    due_before = django_filters.DateFilter(field_name='due_date', lookup_expr='lte')
    due_after  = django_filters.DateFilter(field_name='due_date', lookup_expr='gte')

    # Assignee by UUID or "me" shortcut
    assignee = django_filters.CharFilter(method='filter_assignee')

    # Boolean: overdue tasks
    overdue  = django_filters.BooleanFilter(method='filter_overdue')

    # Unassigned tasks
    unassigned = django_filters.BooleanFilter(method='filter_unassigned')

    class Meta:
        model  = Task
        fields = ['status', 'priority', 'assignee']

    def filter_assignee(self, queryset, name, value):
        if value.lower() == 'me':
            return queryset.filter(assignee=self.request.user)
        return queryset.filter(assignee__id=value)

    def filter_overdue(self, queryset, name, value):
        from django.utils import timezone
        today = timezone.now().date()
        if value:
            return queryset.filter(due_date__lt=today).exclude(status=Task.Status.DONE)
        return queryset.filter(Q(due_date__gte=today) | Q(due_date__isnull=True))

    def filter_unassigned(self, queryset, name, value):
        if value:
            return queryset.filter(assignee__isnull=True)
        return queryset.filter(assignee__isnull=False)

class IsWorkspaceMember(permissions.BasePermission):
    """User must be a member of the workspace to access any resource in it."""
    message = 'You are not a member of this workspace.'

    def has_permission(self, request, view):
        workspace_id = view.kwargs.get('workspace_pk') or request.data.get('workspace')
        if not workspace_id:
            return True
        return Membership.objects.filter(
            workspace_id=workspace_id, user=request.user
        ).exists()

    def has_object_permission(self, request, view, obj):
        # obj is the workspace or a resource belonging to it
        workspace = getattr(obj, 'workspace', obj)
        return Membership.objects.filter(workspace=workspace, user=request.user).exists()


class IsProjectMember(permissions.BasePermission):
    """Read: any workspace member. Write: only project members."""

    def has_object_permission(self, request, view, obj):
        project = getattr(obj, 'project', obj)
        if request.method in permissions.SAFE_METHODS:  # GET, HEAD, OPTIONS
            return True
        return project.members.filter(id=request.user.id).exists()


class IsOwnerOrReadOnly(permissions.BasePermission):
    """Anyone can read. Only the owner/author can write/delete."""

    def has_object_permission(self, request, view, obj):
        if request.method in permissions.SAFE_METHODS:
            return True
        owner = getattr(obj, 'author', None) or getattr(obj, 'owner', None)
        return owner == request.user

    """Only workspace owners or admins can perform admin actions."""

class RequestLoggingMiddleware:
    def __init__(self, get_response):

    def __call__(self, request):

        response = self.get_response(request)
        duration_ms = round((time.time() - start) * 1000, 2)

            'status':   response.status_code,

        })

        # Alert on slow requests (> 2s)
        if duration_ms > 2000:
            logger.warning('slow_request', extra={
                'path': request.path, 'duration': duration_ms
            })

        return response

class JSONFormatter(logging.Formatter):
    """Outputs every log line as JSON — compatible with Datadog, CloudWatch."""

    def format(self, record):
        log_obj = {
            'timestamp': datetime.now(timezone.utc).isoformat(),
            'level':     record.levelname,
            'logger':    record.name,
            'message':   record.getMessage(),
        }
        # Merge any extra= fields from the log call
        for key, val in record.__dict__.items():
            if key not in ('msg', 'args', 'levelname', 'levelno', 'name',
                            'pathname', 'filename', 'module', 'exc_info',
                            'exc_text', 'stack_info', 'lineno', 'funcName',
                            'created', 'msecs', 'relativeCreated', 'thread',
                            'threadName', 'processName', 'process', 'taskName',
                            'message'):
                log_obj[key] = val
        if record.exc_info:
            log_obj['exception'] = self.formatException(record.exc_info)
        return json.dumps(log_obj)

    {"code": "validation_error", "message": "...", "details": {...}}
    """
    response = exception_handler(exc, context)

    if response is not None:
        view = context.get('view')
        logger.warning('api_error', extra={
            'status': response.status_code,
            'exception': exc.__class__.__name__,
            'view': view.__class__.__name__ if view else None,
        })

        error_code = getattr(exc, 'default_code', 'error')
        response.data = {
            'code':    error_code,
            'message': str(exc),
            'details': response.data,
            'status':  response.status_code,
        }

    else:
        # Unhandled exception — log as error
        logger.error('unhandled_exception', exc_info=True, extra={
            'view': context.get('view', {})
        })
        response = Response({
            'code': 'server_error',
            'message': 'An unexpected error occurred.',
        }, status=status.HTTP_500_INTERNAL_SERVER_ERROR)

    return response

@shared_task(bind=True, max_retries=3, default_retry_delay=60)
def send_task_assignment_email(self, task_id: str):
    """
    Send email when a task is assigned.
    Runs async in Celery worker — doesn't block the API response.
    """
    from .models import Task   # import inside to avoid circular imports
    try:
        task = Task.objects.select_related('assignee', 'project', 'reporter').get(id=task_id)
        if not task.assignee or not task.assignee.email:
            return

        send_mail(
            subject=f'[TaskFlow] Task assigned: {task.title}',
            message=f"""
Hi {task.assignee.first_name or task.assignee.username},

You have been assigned a new task in project "{task.project.name}":

  Title:    {task.title}
  Priority: {task.get_priority_display()}
  Due date: {task.due_date or 'Not set'}

View task: https://app.taskflow.io/tasks/{task.id}/

— TaskFlow
            """,
            from_email=settings.DEFAULT_FROM_EMAIL,
            recipient_list=[task.assignee.email],
        )
        logger.info('assignment_email_sent', extra={
            'task_id': task_id,
            'to': task.assignee.email
        })

    except Task.DoesNotExist:
        logger.warning('task_not_found_for_email', extra={'task_id': task_id})
    except Exception as exc:
        logger.error('assignment_email_failed', exc_info=True)
        raise self.retry(exc=exc)   # retry up to max_retries times
